## Directory Purposes

**Repository root (`.py` modules):**
- Purpose: All application source code lives as flat modules here.
- Contains: Entry point, orchestration, UI, I/O, learning logic.
- Key files: `main.py`, `categorizer.py`, `learning_engine.py`, `gui_interface.py`

**`.github/workflows/`:**
- Purpose: CI/CD automation.
- Contains: Conda/pip install, flake8, pytest, PyInstaller artifact upload.
- Key files: `.github/workflows/python-package-conda.yml`

**`原始账单/`:**
- Purpose: Store raw exported bills from WeChat/Alipay before classification.
- Contains: `.csv` / `.xlsx` files with Chinese filenames (person + platform + date range).
- Key files: User-provided; discovered recursively by `data_loader.find_excel_files()`.

**`已分类/`:**
- Purpose: Destination for classified output CSVs (`{person}-{month}-{source}-已分类账单.csv`).
- Contains: Flat or year-subfolder organized CSVs.
- Key files: Generated by `data_exporter.export_to_csv()`; may also land in cwd depending on run directory.

**`分类明细/`:**
- Purpose: Category-split expense/income detail exports (downstream aggregation input).
- Contains: Many `支出-*.csv` / `收入-*.csv` files.
- Key files: Not produced by core pipeline directly — separate user workflow.

**`dist/`:**
- Purpose: PyInstaller build artifacts.
- Contains: `BillCategorizer/` folder with `_internal/` deps and bundled JSON configs.
- Generated: Yes (via `pyinstaller build.spec`).

**`bill/`:**
- Purpose: Local virtual environment (`bill/Lib/site-packages/`).
- Contains: pip-installed packages.
- Generated: Yes — exclude from architecture changes.

**Nested `BillCategorizer/BillCategorizer/`:**
- Purpose: Historical duplicate checkout or sync artifact.
- Contains: Mirror of core modules without latest `learning_engine` enhancements.
- Key files: Do not use for development; edit root-level files only.

## Key File Locations

**Entry Points:**
- `main.py`: Primary application — `python main.py` or `--cli` for command line.
- `WeChatBillCategorizer.py`: Legacy standalone script; not used by `main.py`.
- `bill_analyzer.py`: Standalone CSV analysis — `python bill_analyzer.py`.
- `build.spec`: PyInstaller entry — bundles `main.py` + JSON datas.

**Configuration:**
- `config.py`: `ConfigManager` class with defaults and merge logic.
- `config.json`: User overrides for `categories`, `files`, `limits`, `display`.
- `environment.yml`: Conda dependency specification.
- `build.spec`: Packaged data files list (`config.json`, rules, history).

**Core Logic:**
- `categorizer.py`: `BillCategorizer` — main workflow loop and transaction processing.
- `learning_engine.py`: Rules engine — suggestions, learning, JSON persistence, migration.
- `data_loader.py`: Parsers `_load_wechat_*`, `_load_alipay_*`, `find_excel_files()`.
- `data_exporter.py`: `prepare_final_dataframe()`, `export_to_csv()`.

**Presentation:**
- `user_interface.py`: CLI — ~230 lines, stdin/stdout.
- `gui_interface.py`: GUI — Tkinter modals, transaction window, edit/delete classified rows.

**Testing:**
- `test_gui.py`: Single pytest function `test_gui()`; skipped when `CI=true`.

**Runtime Data (committed in repo):**
- `bill_rules_optimized.json`: Learned merchant|product → category mappings.
- `bill_history.json`: Append-only classification audit log.

## Naming Conventions

**Files:**
- Python modules: `snake_case.py` at repository root (e.g., `data_loader.py`, `learning_engine.py`).
- Config/data: `snake_case.json` (e.g., `bill_rules_optimized.json`, `config.json`).
- User bill CSVs: `{person}-{month}-{platform}-已分类账单.csv` (Chinese month label, e.g., `杜雨秦-4月-支付宝-已分类账单.csv`).
- Raw bills: `{person}-{platform}交易明细(...).csv` or `{person}-{platform}-{month}.csv`.

**Directories:**
- Chinese descriptive names for user data: `原始账单/`, `已分类/`, `分类明细/`.
- Standard tooling dirs: `.github/`, `dist/`, lowercase English.

**Classes:**
- PascalCase: `BillCategorizer`, `ConfigManager`, `DataLoader`, `LearningEngine`, `DataExporter`, `UserInterface`, `GUIInterface`, `BillAnalyzer`.

**Functions / methods:**
- snake_case: `load_excel_file`, `get_suggestions`, `learn_from_decision`, `_process_single_transaction`.
- Private helpers: leading underscore (`_load_alipay_csv`, `_migrate_rules`, `_standardize_to_wechat_format`).

**Variables:**
- snake_case instance attrs: `self.current_bill_source`, `self.merchant_index`, `self.current_processed_df`.
- DataFrame columns (internal): Chinese names (`交易对方`, `分类`, `人员`); export columns English (`Name`, `Category`, `Amount`).

## Where to Add New Code

**New bill source (e.g., bank CSV format):**
- Primary code: `data_loader.py` — add `_load_{source}_csv` / `_load_{source}_excel`, branch in `load_excel_file()`.
- Config: `config.py` default `categories.bill_sources` and/or `config.json`.
- Tests: No existing unit test dir — add `tests/test_data_loader.py` if introducing pytest coverage (pattern from `test_gui.py` at root).

**New classification / learning behavior:**
- Primary code: `learning_engine.py` — `get_suggestions()`, `learn_from_decision()`.
- Orchestration hooks: `categorizer.py` `_process_single_transaction()` if auto-apply rules change.

**New UI interaction (shared CLI + GUI):**
- CLI: `user_interface.py` — new method matching GUI counterpart.
- GUI: `gui_interface.py` — implement same method signature; use `threading.Event` + `user_choice` for blocking dialogs.
- Detection: If GUI-only feature, guard with `hasattr(self.ui, 'show_results')` in `categorizer.py` (existing pattern) or refactor to explicit mode flag.

**New export column or format:**
- Primary code: `data_exporter.py` — `prepare_final_dataframe()`, optionally `export_to_csv()` filename logic.

**New configuration key:**
- Defaults: `config.py` → `ConfigManager.default_config`.
- Override: document in `config.json`; access via `config.get("section.key")`.

**Standalone utility (like analysis):**
- New top-level script: e.g., `my_tool.py` with `if __name__ == "__main__"` — keep independent of `BillCategorizer` unless integration needed.

**PyInstaller packaging change:**
- Edit: `build.spec` — `datas`, `hiddenimports`, `console` flag.
